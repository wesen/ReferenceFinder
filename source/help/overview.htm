Overview

This page gives a brief overview of ReferenceFinder to get you started. The screen images shown are from the Macintosh version of ReferenceFinder, but windows on other platforms should look similar.

When ReferenceFinder first starts up, it will display its main window while it constructs its database of points and lines. This initialization will take a few seconds to a few minutes (depending on the speed and memory of your machine), but ReferenceFinder will regularly update the progress as it works.


Figure O-1. The main window after startup while it initializes.

When initialization is complete, ReferenceFinder will display "Ready" and the number of points (marks) and lines that it has constructed in preparation for your search.


Figure O-2. The main window after initialization is complete.

The main window contains a few controls that let you choose what you are looking for. The two radio buttons, Point and Line, select whether you're seeking to create a point or a fold line. If you select Point, you will enter the x and y coordinates of the point in the adjacent text boxes. If you select Line, then you will need to enter x and y coordinates for two distinct points along the line.

The default paper size is a unit square, so all coordinates values should be between 0 and 1. (You can change the paper size in the Preferences if, for example, you're using A4 paper.) You can enter coordinates either as their decimal values (e.g., 0.4142), or as a mathematical expression (e.g., sqrt(2)-1, or even d-1). Expressions may contain any of the operators +, -, *, /, ^, (), sqrt, sin, cos, tan, deg2rad and constants e, pi, phi, Phi, and d, which is an abbreviation for sqrt(2) (which shows up a lot in origami crease patterns). Trigonometric functions use radians. Function deg2rad converts degrees to radians. If you enter an expression that ReferenceFinder can't interpret, or that interprets to a point outside the paper, ReferenceFinder will let you know so that you can take corrective action.

Once you have entered the coordinate values for one or two points, you can select Action->Get Reference or simply hit the Return or Enter key. ReferenceFinder immediately finds several folding sequences that give the desired point or line and displays folding diagrams and verbal directions for them in the window. At this point, the File->Print... command will be enabled, and you can print the resulting diagrams for a permanent record.


Figure O-3. Some solutions found by ReferenceFinder.

For each sequence, ReferenceFinder displays the error between the displayed sequence and your chosen target, and the rank, which is the number of folds needed to create the mark or line from a blank paper. Sequences with smaller rank are shown first; within a given rank, the lowest error is shown first.

That's all there is to it. If you want to find another point or line, enter new coordinates and hit Return.

There are several settings you can adjust within ReferenceFinder that affect the screen display and the database of references; these let you set the paper dimensions, trade off initialization time against accuracy, and even choose which of the Huzita-Hatori axioms to use in constructing references. All of these settings are accessible through the Preferences command, which is described later in this documentation. At any point, you can refer to the Help menu for explanations of menu commands, windows, and possible actions.
